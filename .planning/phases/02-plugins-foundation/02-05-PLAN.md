<objective>
Fix 3 issues identified in cross-AI code review (02-REVIEWS.md):

1. **HttpClient/Singleton anti-pattern** — Replace raw HttpClient with IHttpClientFactory
2. **Culture-dependent URL formatting** — Use CultureInfo.InvariantCulture for lat/lon
3. **Null input guard** — Add null check on iataCode to prevent NullReferenceException

Output: Hardened plugin code with proper HttpClient lifecycle, culture-safe formatting, and defensive input validation.
</objective>

<tasks>

## Task 1: Fix WeatherPlugin HttpClient lifecycle

Replace `HttpClient` constructor parameter with `IHttpClientFactory`.
Create client per-call via `_httpClientFactory.CreateClient()`.

Update `DependencyInjection.cs`:
- Add `services.AddHttpClient<Plugins.WeatherPlugin>()` (typed client registration)
- Change from `AddSingleton` to let the HttpClient factory manage lifetime

## Task 2: Fix culture-dependent URL formatting

In `FetchWeatherAsync`, use `FormattableString.Invariant()` or explicit `CultureInfo.InvariantCulture` for the latitude/longitude interpolation in the Open-Meteo URL.

## Task 3: Add null guards on iataCode

In both `AirportPlugin.GetAirportInfoAsync` and `WeatherPlugin.GetAirportWeatherAsync`:
- Return null early if iataCode is null or whitespace
- Prevents NullReferenceException from `.ToUpperInvariant()` and dictionary lookup

</tasks>
